= 在NetApp Workload Factory中追蹤資源成本
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
使用NetApp Workload Factory以統一視圖追蹤 FSx for ONTAP檔案系統的成本和使用情況。成本數據有助於您有效管理預算和優化資源。  AWS Cost Explorer 提供成本資料。

.關於這項工作
使用下列權限從 AWS Cost Explorer 提取 FSx for ONTAP檔案系統資源的成本和使用情況資料：

* `ce:GetCostAndUsage`
* `ce:GetTags`


.開始之前
link:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["使用_檢視、規劃和分析_權限策略授予憑證"^]在 Workload Factory 中追蹤 FSx 的ONTAP成本。

.步驟
. 使用其中一項登link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["主控台體驗"^]入。
. 選擇選單 image:workloads-menu-icon.png["漢堡選單圖示用於導航至儲存、EDA、AI、資料庫、VMware 和管理等工作負載。"] 然後選擇“儲存”。
. 從儲存選單中，選擇“成本”。
. 在「成本」頁面上，透過提供以下資訊來篩選 FSx for ONTAP資源的成本和容量資料：
+
.. *AWS 帳戶*：選擇要檢視成本資料的帳戶。
.. *憑證*：選擇具有_檢視、規劃和分析_權限的憑證。
.. *區域*：選擇要查看成本資料的 AWS 區域。
.. *成本資料範圍*：選擇要查看的成本資料的時間範圍。


. 查看 FSx for ONTAP資源的*成本詳情*。
