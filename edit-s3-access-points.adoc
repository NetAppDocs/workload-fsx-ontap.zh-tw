= 在 NetApp Workload Factory 中編輯某個 volume 的 S3 存取端點
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
在 NetApp Workload Factory 中更新 S3 存取點詳細資訊並管理磁碟區的標籤。

您也可以啟用或停用存取點的 *Inventory table* 。Inventory table 會為所有可透過 S3 存取點存取的物件產生中繼資料，並產生 AWS S3 請求費用。如需詳細資訊，請參閱 link:https://aws.amazon.com/s3/pricing/["Amazon S3 定價文件"^] 。

.步驟
. 使用其中一項登link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["主控台體驗"^]入。
. 選擇選單 image:workloads-menu-icon.png["漢堡選單圖示用於導航至儲存、EDA、AI、資料庫、VMware 和管理等工作負載。"] 然後選擇“儲存”。
. 從儲存選單中，選擇 *FSx for ONTAP*。
. 從 *FSx for ONTAP* 中選擇要更新的磁碟區的檔案系統的操作選單，然後選擇 *管理*。
. 從檔案系統總覽中、選取 * Volumes （磁碟區） * 標籤。
. 在「磁碟區」標籤中、選取您要編輯 S3 存取點之磁碟區的動作功能表、然後選取 *進階動作*、接著選取 *管理 S3 存取點*。
. 從 *Manage S3 access points* 畫面中，選擇操作選單，然後選擇 *Edit access point* 。
. 進行更新，然後選取 *Apply* 。
+
若要設定 journal table 功能的基礎架構，您必須選擇 *Internet* 作為 S3 存取點的網路設定。link:setup-journal-table.html["瞭解如何為 NetApp Workload Factory 設定 Journal 資料表基礎架構"]。
